Basic Commands

Essential Git commands every developer must know. These form the foundation of your Git workflow.

1
git init
Initializes a new Git repository. Takes your project under version control and creates a .git folder.
Usage: Run in your new project folder
2
git clone <repository-url>
Copies a repository from remote server to your local machine. First step to start working on a project.
Example: git clone https://github.com/user/project.git
3
git add <file> | git add .
Adds files to the staging area. Determines which changes will be included in the next commit.
All files: git add .
Single file: git add index.html
4
git commit -m "commit message"
Permanently saves changes from staging area to repository. Each commit is a snapshot in time.
Good message: "Add user authentication feature"
5
git status
Shows the current state of working directory. Which files changed, which are staged.
Usage: Check status before every operation
6
git log --oneline --graph
Displays commit history in a visual way. Track your project's evolution over time.
Detailed: git log --all --decorate --oneline --graph

Branch Management

Master branching strategies for efficient feature development and collaboration.

7
git branch <branch-name>
Creates a new branch. Use separate branches for feature development.
Naming: feature/user-login, bugfix/payment-error
8
git checkout <branch>
Switches between different branches. In modern Git, you can also use git switch.
Create + switch: git checkout -b feature/new-feature
9
git merge <branch-name>
Merges one branch into current branch. Merge to main after feature completion.
Workflow: git checkout main && git merge feature/login
10
git branch -d <branch>
Deletes merged branches. Remove unnecessary branches for clean repository.
Force delete: git branch -D branch-name

Remote Operations

Collaborate effectively with team members using remote repositories.

11
git push origin <branch>
Sends local commits to remote server. Share your work with the team.
First push: git push -u origin main
12
git pull origin <branch>
Fetches changes from remote server to local repository. Stay up to date.
With rebase: git pull --rebase origin main
13
git fetch
Downloads changes from remote but doesn't merge. Safe way to update.
Then merge: git fetch && git merge origin/main
14
git remote -v
Lists connected remote repositories. Which servers are you connected to?
Add remote: git remote add upstream https://github.com/original/repo.git

Git Workflow & Behind the Scenes

Understand the complete Git workflow and what happens behind the scenes

1
Clone Repository
git clone <repository-url>
Downloads the entire project history from remote server to your local machine. Creates a complete copy of the repository including all branches, commits, and files. This establishes the connection between your local and remote repositories.
2
Create Branch
git checkout -b feature/new-feature
Creates a new branch pointer that references the current commit. Branches in Git are lightweight movable pointers to specific commits. This allows you to work on features in isolation without affecting the main codebase.
3
Make Changes
vim index.html
Modify files in your working directory. Git tracks these changes but doesn't save them yet. The working directory is where you actually edit files - Git monitors what's different from the last commit.
4
Stage Changes
git add .
Moves changes from working directory to staging area (index). The staging area is a file in .git/index that stores information about what will go into your next commit. This step lets you selectively choose which changes to include.
5
Commit Changes
git commit -m "Add new feature"
Creates a snapshot of staged changes in Git's object database. Each commit contains tree objects (directories), blob objects (files), author information, timestamp, and a reference to parent commit(s). This creates a permanent record in project history.
6
Push to Remote
git push origin feature/new-feature
Uploads your local commits to the remote repository. This transfers all the objects (commits, trees, blobs) that don't exist on the remote server. Makes your changes available to other team members and creates a backup of your work.

Git's Three-Tree Architecture

Working Directory: Your actual files where you make changes. Git tracks modifications here.

Staging Area (Index): A binary file in .git/index that stores information about what will go into your next commit.

Repository (.git directory): Contains object database with compressed snapshots of your project.

Object Database: Stores commits (snapshots), trees (directories), blobs (files), and tags using SHA-1 hashes.

Advanced Commands

Master advanced Git techniques for complex workflows and repository management.

15
git rebase -i HEAD~3
Interactive rebase to edit, squash, or reorder commits. Clean up commit history.
Operations: pick, squash, edit, drop commits
16
git stash
Temporarily saves uncommitted changes. Switch branches without losing work.
Restore: git stash pop
17
git cherry-pick <commit>
Applies specific commits to current branch. Move commits between branches.
Range: git cherry-pick commit1..commit3
18
git reset --hard HEAD~1
Completely removes last commit. WARNING: Destructive operation!
Safer: git reset --soft HEAD~1

Best Practices

Professional Git practices for clean, maintainable repositories

Clear Commit Messages

Write descriptive, actionable commit messages that explain what and why. Use present tense and be specific about changes.

Atomic Commits

Make small, focused commits that do one thing well. Each commit should represent a logical unit of change.

Feature Branches

Use separate branches for features, bugs, and experiments. Keep main branch stable and deployable.

Protect Sensitive Data

Never commit passwords, keys, or sensitive information. Use .gitignore and environment variables.
